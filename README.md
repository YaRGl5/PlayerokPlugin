⚙️ Архитектура и принцип работы (State Machine)
Наши плагины подключаются к ядру бота и используют концепцию конечного автомата (State Machine). Это означает, что для каждой сделки плагин ведет диалог с покупателем по строго заданным "шагам" (состояниям), не путая данные разных покупателей даже при высокой нагрузке.

Как это работает:

[NEW_DEAL] Поступает новый заказ. Бот пишет покупателю "Привет! Пришлите данные..." и переходит в статус WAITING_FOR_DATA.
[NEW_MESSAGE] Покупатель присылает данные. Бот переводит сделку в статус PROCESSING, обращается к нужным API (SMM-панели, поставщики баланса).
[SUCCESS / ERROR] При успешном исполнении бот переводит сделку в COMPLETED, благодарит покупателя и закрывает ее на сайте. В случае ошибки сообщает статус покупателю или передает спор администратору.
Такая архитектура позволила сделать бота легко масштабируемым. Вы просто добавляете новый плагин с новой логикой шагов — и он работает параллельно с остальными!

🧩 Обзор доступных плагинов (Модулей)
1. 💌 Модуль vouchers (Цифровые подарочные карты)
Назначение: Автоматическая закупка и выдача ключей активации (Apple Gift Card, Google Play, Steam Wallet и др.).

Принцип работы: После покупки бот автоматически закупает нужный ключ у сторонних поставщиков через их API, либо извлекает из локальной базы, и моментально отправляет его покупателю. Это позволяет продавать ваучеры без необходимости держать огромный склад ключей заранее.
Поддерживает автоматическое перевыставление товара на Playerok (чтобы лот не пропадал с витрины), а также работает с несколькими номиналами через алиасы названий.
2. 👍 Модуль 
auto_smm
 (Авто-накрутка в соц. сетях)
Назначение: Полностью автоматизированная продажа лайков, подписчиков и просмотров.

Принцип работы: При покупке лота бот запрашивает у клиента ссылку (например, на пост в Telegram или TikTok). Получив её (state: WAITING_FOR_LINK), бот делает запрос по API к выбранной SMM-панели, оплачивает накрутку и рапортует об успехе.
Встроен парсер для автоматического извлечения количества услуг из названия лота.
3. ⭐️ Модуль auto_stars (Telegram Premium Stars)
Назначение: Пополнение звездами Telegram.

Принцип работы: Запрашивает у клиента логин/тег Telegram (@username). Бот подключается к Fragment API (или аналогу), проверяет существование пользователя, покупает необходимое количество звезд (определяется из лота) и закрывает заказ.
4. 🎮 Модуль steam_handler (Автопополнение Steam)
Назначение: Пополнение кошелька Steam по логину аккаунта.

Принцип работы: Работает через API поставщика DesslyHub. Запрашивает логин Steam в чате, делает API-запрос check_login для проверки доступности аккаунта (исключает ошибки с закрытыми регионами), а затем отправляет запрос 
refill
 на нужную сумму. В случае неудачи (аккаунт не найден) бот просит покупателя прислать корректный логин.
5. 🤖 Модуль gpt_consultant (AI Служба поддержки)
Назначение: Умный AI-консультант в чате Playerok на базе Groq API (семейство моделей Llama), работающий в роли первой линии поддержки магазина.

Как это работает под капотом:

Контекст и База Данных: Бот имеет доступ к knowledge_base.json (информация об актуальных товарах, ценах, правилах магазина и FAQ). Перед каждым ответом бот подмешивает эту информацию в System Prompt, поэтому AI "знает", что он продаёт и почем.
Классификация намерений (Intent Classification): Входящее сообщение от покупателя сначала классифицируется. Если это простой вопрос ("Как активировать код?") — бот отвечает сам.
Строгие правила: AI ограничен жестким набором директив (System Prompt). Например: отвечать кратко (1-3 предложения), не давать скидок без промокода, не обсуждать конкурентов.
Умная Эскалация (Перехват): Если покупатель пишет "Код не работает", просит возврат, или нейросеть понимает, что случай сложный — классификатор автоматически останавливает генерацию ответа от AI, пишет покупателю "Перевожу на специалиста..." и моментально отправляет уведомление живому продавцу в Telegram.
История диалогов: Бот хранит контекст беседы для каждого покупателя (до N последних сообщений), чтобы поддерживать естественный и связный диалог.
🗂️ Как добавить этот проект к себе
Бот легко разворачивается на любом VPS с поддержкой Python. В каждом модуле реализовано логирование (logging) и система автоматического перезапуска запросов (failover retry) при сбоях на самой платформе Playerok.

Внимание: Для работы вам потребуются API-ключи от соответствующих сервисов (поставщиков баланса, SMM-панелей, Groq для консультанта).
